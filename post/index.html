Git Note

2021年3月5日
Git是一个分布式的版本管理器（是一个软件） Git跟踪的是修改，而不是文件 Git名词 仓库 就是一个包含.git文件夹的文件夹 工作目录 打开文件管理器，进入/home/user目录，那么此时user，就是你的工作目录；如果你进入了/home/user/document目录，此时你的工作目录则变更为document。总的来说，就是你在哪个目录，哪个目录就是你的工作目录。 Git分为三棵树： 工作目录 暂存区 HEAD Git命令 init 在一个空文件夹下执行git init，会将此文件夹初始化为git仓库 如果要指定文件夹初始化为仓库，则执行git init 。 clone 克隆一个仓库到工作目录下，git clone <地址> add 将工作目录中的test.txt文件添加到暂存区，git add test.txt commit 将暂存区内test.txt文件提交到HEAD中（此时暂存区内只有test.txt一个文件），git commit -m “对此次提交的注释” -m 选项是对后面的注释添加的参数。 如果不写 -m，只写git commit，则会在此句命令后，系统会提示让你填写注释。 如果想将暂存区内所有的文件都提交到HEAD中，git commit -a -m “对此次提交的注释”……
