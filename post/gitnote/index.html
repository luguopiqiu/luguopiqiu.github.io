Git Note

2021年3月5日

Git是一个分布式的版本管理器（是一个软件） Git跟踪的是修改，而不是文件

Git名词

  • 仓库 就是一个包含.git文件夹的文件夹
  • 工作目录 打开文件管理器，进入/home/user目录，那么此时user，就是你的工作目录；如果你进入了/home/user/document目录，此时你的工作目录则变更为document。总的来说，就是你在哪个目录，哪个目录就是你的工作目录。

Git分为三棵树：

  1. 工作目录
  2. 暂存区
  3. HEAD

Git命令

  • init

在一个空文件夹下执行git init，会将此文件夹初始化为git仓库 如果要指定文件夹初始化为仓库，则执行git init 。

  • clone

克隆一个仓库到工作目录下，git clone <地址>

  • add

将工作目录中的test.txt文件添加到暂存区，git add test.txt

  • commit

将暂存区内test.txt文件提交到HEAD中（此时暂存区内只有test.txt一个文件），git commit -m “对此次提交的注释”

-m 选项是对后面的注释添加的参数。 如果不写 -m，只写git commit，则会在此句命令后，系统会提示让你填写注释。 如果想将暂存区内所有的文件都提交到HEAD中，git commit -a -m “对此次提交的注释” -a选项只会提交文件的修改，不会添加新的文件到HEAD中

  • status

查看暂存区的状态，命令git status

  • diff

查看指定文件在“工作区”与“版本库”中的不同之处，git diff

  • log

显示从最近到最远的提交日志，git log。 添加 –pretty=oneline 参数：只显示“快照哈希值”与“提交注释”，git log –pretty=oneline。

  • reset

回退到以前的版本，git reset –hard <快照哈希值> 撤销添加到暂存区的动作，git reset HEAD

  • reflog

查看你的每一次命令，git reflog。

  • checkout

用版本库中的文件替换工作区中的文件 让文件回到最近一次操作的状态(commit或add)，git checkout –

创建分支并切换到新分支上，git checkout -b 。(git switch -c 。这是最新版本添加的命令)

切换分支，git checkout 。(git switch 。这是最新版本添加的命令)

在本地创建和远程分支对应的分支，git checkout -b <branchorigin/。

  • branch

列出所有分支，当前所在分支前有 * 号。git branch。 删除分支，git branch -d 。

建立本地分支和远程分支的关联，git branch –set-upstream 。

  • switch (最新版本添加的命令)

  • merge

将分支的内容合并到当前分支上，git merge 。

禁用Fast forward合并，git merge –no-ff -m “对此次合并的注释”。(此次合并会创建一个新的commit)

  • rm

从版本库中删除文件，git rm 。(删除版本库文件前需要删除工作区的文件)

  • remote

关联一个远程库，git remote add origin <ssh地址/http地址>

查看远程库的信息，git remote。

查看远程库的详细信息， git remote -v。

  • push

当前分支master推送到远程，git push origin master。 加上 -u 参数，还会把远程master与本地master关联起来，下次推送或拉取时就可以简化命令，git push -u orgin master。

  • pull

拉取远程库的分支内容与本地的分支内容合并。

如果你想将本地仓库提交到其他地址，则执行git pull <地址>

  • config

关于git配置文件的命令

查看配置信息，命令git config –list

  • stash

把当前工作现场“储存”起来，储存到当前分支上。git stash。

查看当前分支的存储的“stash”，git stash list。

恢复“工作现场”，git stash apply。这个恢复不会把储存的“stash”删除，git stash drop，后就能删除。

git stash pop。这个命令能恢复“工作现场”，同时删除“stash”删除。

  • cherry-pick

指定一个提交合并到当前分支，git cherry-pick <哈希值>。

多人协作工作模式

  1. 首先，可以试图用git push origin 推送自己的修改；

  2. 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；

  3. 如果合并有冲突，则解决冲突，并在本地提交；

  4. 没有冲突或者解决掉冲突后，再用git push origin 推送就能成功！

如果git pull提示no tracking information，则说明本地分支和远程分支的链接关系没有创建，用命令git branch –set-upstream-to <branchorigin/。
